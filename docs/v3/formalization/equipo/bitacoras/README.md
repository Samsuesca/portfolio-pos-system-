# Bitácoras del equipo UCR

> **Última actualización:** 2026-05-16
> **Contexto:** instrumentación operativa del [equipo-roadmap-2026.md](../equipo-roadmap-2026.md).

---

## Propósito

Una bitácora por persona del equipo UCR. Documento vivo que:

1. Define el rol target dentro de UCR.
2. Mapea las habilidades requeridas y el nivel actual vs el target.
3. Estructura la ruta de aprendizaje (cursos Platzi + recursos complementarios).
4. Lista mini-proyectos aplicados al negocio que sirven de evidencia.
5. Registra mes a mes el progreso, los pagos y los ajustes acordados.
6. Es input para la revisión semestral y para la activación del plan de equity.

---

## Estructura común

Todas las bitácoras siguen la misma plantilla:

```markdown
# Bitácora — Nombre

> Última actualización · Track · Fecha de inicio formal

## 1. Perfil base
## 2. Rol target
## 3. Ruta de habilidades (radar 1-5)
## 4. Plan de estudio próximos 6 meses
## 5. Mini-proyectos aplicados (backlog)
## 6. Compensación
## 7. Registro mensual
## 8. Revisiones semestrales
## 9. Notas y acuerdos
```

## Reglas de mantenimiento

- **Quien actualiza:** cada persona, con review mensual de Angel.
- **Frecuencia mínima:** entrada en §7 (registro mensual) el último día hábil del mes.
- **Revisión semestral:** sesión 1-1 de 90 min con Angel + actualización formal de §3 y §8.
- **Privacidad:** son documentos internos del equipo. No se publican a clientes ni se referencian en marketing.
- **Conflicto:** si la realidad operativa diverge del plan (la persona cambia de área, asume responsabilidades distintas, etc.), se actualiza la bitácora — no se ignora. La bitácora es el contrato vivo.
